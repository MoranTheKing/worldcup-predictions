# World Cup Predictions Handoff

תאריך: 2026-04-21  
Branch פעיל: `master`  
Repo: `https://github.com/MoranTheKing/worldcup-predictions`

## תמונת מצב מהירה

האפליקציה היא PWA לתחזיות מונדיאל 2026 עם Next.js 16, Supabase ו-RTL עברי.
יש שכבת Auth, Onboarding, Profile, Tournament UI, ותחילת ארכיטקטורה לליגות/משחקים.

בפועל, עמוד הטורניר כבר לא placeholder:
- הוא קורא נתוני `teams` ו-`matches` מ-Supabase
- הוא ממיין לפי שוברי שוויון של FIFA
- הוא מציג סטטוסי העפלה/הדחה רק כשיש ודאות מתמטית
- הוא כולל טבלת Best 3rd Place עם certainty אמיתי

## מה הושלם עד עכשיו

### תשתית ו-UI
- הוקם פרויקט Next.js 16 + Tailwind v4 + TypeScript
- הוחל RTL מלא עם `lang="he"` ו-`dir="rtl"`
- הוטמע design system בסגנון WC2026
- `DashboardShell` כבר מציב sidebar בצד ימין בדסקטופ
- במסכים ניידים יש header עליון ו-bottom nav

### Auth / User Flow
- קיימים `login`, `signup`, `auth callback`
- יש clients ל-Supabase בדפדפן ובשרת
- יש הגנת נתיבים דרך `middleware.ts`
- נוסף גם `proxy.ts` מותאם ל-Next 16, אבל כרגע `middleware.ts` עדיין קיים ולכן נשארת אזהרת deprecation
- `onboarding` קיים ועובד עם username + winner + top scorer
- `profile` קיים עם inline editing ונעילה לפי kickoff של המשחק הראשון

### DB / Supabase
- קיימות migrations 1-4 עבור schema בסיסי, auth/RLS, i18n לקבוצות, ותיקוני policies
- קיימת migration 5:
  - `supabase/migrations/20260421000005_add_team_standings_cache.sql`
  - מוסיפה ל-`teams`:
    - `points`
    - `goals_for`
    - `goals_against`
    - `fair_play_score`
    - `fifa_ranking`
    - default תקין ל-`is_eliminated`
- נוספה migration 6:
  - `supabase/migrations/20260421000006_expand_team_standings_cache.sql`
  - מוודאת/מנרמלת את כל שדות ה-cache הנדרשים:
    - `fifa_ranking`
    - `points`
    - `goals_for`
    - `goals_against`
    - `fair_play_score`
    - `played_count`
    - `is_eliminated`
  - מוסיפה defaults, backfill בסיסי, ו-`NOT NULL`

### Seeding / Data
- קיים `scripts/seed-teams.ts`
- קיים `scripts/seed-players-mock.ts`
- קיים `scripts/fetch-players.ts`
- נוסף `scripts/seed-fifa-rankings.ts`
- נוסף script ב-`package.json`:
  - `npm run seed:rankings`
- ה-seed של דירוגי FIFA כבר הורץ בהצלחה על ה-DB הקיים:
  - `Updated FIFA rankings for 48 teams.`

### Tournament Logic
- `lib/utils/standings.ts` עודכן משמעותית
- סדר שוברי השוויון הפעיל כרגע:
  1. H2H Points
  2. H2H Goal Difference
  3. H2H Goals Scored
  4. Overall Goal Difference
  5. Overall Goals Scored
  6. Fair Play
  7. FIFA Ranking
- בטבלת מקום 3 אין H2H, רק השוואה כללית בין בתים, כפי שצריך
- נוספה שכבת mathematical certainty:
  - לכל בית מחושבים תרחישי נקודות אפשריים לפי המשחקים שעוד לא הוכרעו
  - `qualified` מוצג רק אם מקום בטופ 2 מובטח מתמטית
  - `eliminated` מוצג רק אם אין יותר מסלול ריאלי להעפלה
  - `third_qualified` מוצג רק אם נבחרת לא יכולה לרדת מחוץ לטופ 8 של מקומות 3
  - אחרת מוצג `pending`

### Tournament UI
- `app/dashboard/tournament/page.tsx` כבר קורא:
  - `points`
  - `goals_for`
  - `goals_against`
  - `fair_play_score`
  - `fifa_ranking`
  - `played_count`
  - `is_eliminated`
- יש fallback חלקי אם חלק משדות ה-cache עדיין לא קיימים/לא מאוכלסים
- `TournamentClient.tsx` עודכן כך ש:
  - אין יותר תוויות qualification לפי rank בלבד
  - טבלת Best 3rd מציגה את כל 12 המקומות השלישיים עם certainty אמיתי
  - ה-counter נשאר `נבחרות שנותרו`
  - `teamsRemaining` מחושב בפועל כ-`teams.length - eliminatedCount`

## מצב בדיקות

- `npm run lint` עובר
- `npm run seed:rankings` עובר
- `npm run dev` עולה, אבל כבר רץ dev server אחר על `3000`

## דברים חשובים במיוחד לשיחה/מחשב הבא

### 1. למשוך את הקוד
```bash
git checkout master
git pull origin master
```

### 2. להתקין תלויות
```bash
npm install
```

### 3. להחזיר `.env.local`
הקובץ לא נשמר ב-Git. נדרשים לפחות:
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `API_FOOTBALL_KEY`

### 4. להריץ migration 6 ב-Supabase
להחיל את:
`supabase/migrations/20260421000006_expand_team_standings_cache.sql`

### 5. אם צריך לזרוע דירוגי FIFA מחדש
```bash
npm run seed:rankings
```

### 6. נקודת תשומת לב ב-Next 16
כרגע קיימים גם `middleware.ts` וגם `proxy.ts`.
כדי להפסיק את אזהרת ה-deprecation צריך להחליט אם למחוק את `middleware.ts` ולעבור רשמית רק ל-`proxy.ts`.

## שורה תחתונה

המערכת כבר במקום טוב מאוד מבחינת Tournament UX ו-logic:
- ה-standings מחושבים
- tie-breakers מסודרים
- certainty מחושב
- FIFA rankings נוספו

החסר המרכזי עכשיו הוא לא עיצוב, אלא data pipeline אמיתי למשחקים, bets, ו-league scoring.
