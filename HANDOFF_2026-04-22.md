# Handoff - 2026-04-22

## Product state

By 2026-04-22, the app was in the middle of the social predictions phase:

- global auth wrapped the app
- `/game` served as the protected social predictions hub
- private leagues supported create, join, leave, remove member, and delete
- opponent view already hid scheduled matches and unresolved placeholders
- the UI had landed total hits, joker split, and premium locked-outright badges

## Important update from the 2026-04-23 audit

Several security issues were found in the social layer after this handoff:

- permissive SELECT RLS on social prediction tables
- missing DB-level kickoff locks
- pre-kickoff outright serialization leaks
- direct join-page invite-code enumeration
- open redirects in auth entry points
- remotely reachable dev tooling in non-production

Those fixes are now documented in:

- `SECURITY_AUDIT_2026-04-23.md`

And require these new migrations:

- `20260423000018_restore_social_prediction_privacy.sql`
- `20260423000019_enforce_prediction_lock_windows.sql`

## Additional product update from 2026-04-23

The auth flow no longer drops fresh users into the game anonymously:

- a dedicated `/onboarding` flow now runs after login/signup until setup is complete
- each player must choose a unique nickname before reaching `/game` or `/dashboard`
- pre-kickoff onboarding also captures winner and top-scorer picks
- avatar selection is optional and supports default built-in avatars plus the current Google avatar when present
- nickname uniqueness is enforced in the database by `20260423000020_enforce_unique_profile_handles.sql`
- provider-prefilled users still start on the profile step first so they can edit before moving on

The predictions hub also received a UI cleanup on 2026-04-23:

- the outright-picks block on `/game/predictions` was simplified to reduce redundant copy
- save controls and current outright selections now sit in a tighter, more visual layout
- the editable state no longer repeats the same winner/top-scorer values twice on screen
