## Phase 2: League Debug + Full Predictions Hub

Date: 2026-04-22

### Bug Fix: Create League Failing Silently

Root cause: `leagues.owner_id` had TWO foreign key constraints:
1. Original init FK → `public.users(id)` (old table)
2. Phase 1 FK → `public.profiles(id)` (new table)

New users (Google OAuth) may not exist in `public.users`, causing the insert to fail
violating the original FK. The error was swallowed by the generic error path.

Fixes applied:
- **Migration `20260422000011_fix_leagues_owner_fk.sql`**: drops `leagues_owner_id_fkey`
  (old FK to `users`), backfills `public.users` from `auth.users`, ensures the Phase-1
  profiles FK exists.
- **`app/actions/league.ts` rewrite**: full `try/catch` on every DB call, `console.error`
  with the actual error message, admin client used for the `leagues` insert (bypasses RLS
  edge-cases while auth is still verified via `getUser()`).

`app/game/layout.tsx` — updated to render `GameSubNav` above children, with shared page

### My Leagues UI (`/game/leagues`)

### My Predictions UI (`/game/predictions`)

#### Match Betting

- `app/game/predictions/page.tsx` — server: fetches `matches WHERE status='scheduled'`,
  teams (via `attachTeamsToMatches`), and existing `predictions` for the user
- `app/game/predictions/MatchPredictionCard.tsx` — client card per match:
  - Home/away team logos, Hebrew names, stage label, date/time
  - Two score number inputs (home, away), existing prediction pre-filled as `defaultValue`
  - `useActionState` with `upsertMatchPrediction.bind(null, matchId)` pattern
  - Per-card spinner, inline error, "✓ הניחוש נשמר" success feedback
- `app/game/predictions/PredictionsClient.tsx` — renders all cards + gates unauthenticated users

#### Tournament Outrights

- `app/game/predictions/OutrightForm.tsx` — client form:
  - `<select>` dropdown for winner team (all teams, sorted by `name_he`)
  - Text input for top scorer
  - Pre-populated from existing `tournament_predictions` row
  - `useActionState` with `upsertTournamentPrediction`

### Server Actions: `app/actions/predictions.ts`

`upsertMatchPrediction(matchId, _prev, formData)`:
- Validates scores (non-negative integers)
- Upserts into `predictions` table with `onConflict: "user_id,match_id"`

`upsertTournamentPrediction(_prev, formData)`:
- Upserts into `tournament_predictions` (new scoring table)
- Mirrors to `outright_bets` (backward compat with ProfileClient)
- `predicted_winner_team_id` cast to integer (teams.id is BIGINT)

## Phase 2.1 - Security Hardening + League Management UX

Date: 2026-04-22

This pass tightened both the DB model and the `/game` UX after the first Phase 2 rollout.

### Security / DB

- added `supabase/migrations/20260422000014_harden_game_social_security.sql`
- `generate_invite_code()` now creates `4`-character uppercase alphanumeric invite codes
- legacy invite codes that do not match the new format are regenerated during the migration
- added `league_join_attempts` for per-user brute-force throttling on invalid invite-code submissions
- reinforced flat `SELECT USING (true)` policies on:
  `league_members`, `predictions`, `tournament_predictions`

### Server Actions

- `app/actions/league.ts` now includes:
  `createLeague`
  `joinLeague` with rate-limit guard
  `leaveLeague`
  `removeMember`
  `deleteLeague`
- owner-only actions now verify `auth.uid() === owner_id` server-side
- leave flow verifies the authenticated user matches the target user row before deleting membership

### UI / Routes

- `/game/leagues` now uses inline Create / Join panels instead of exposing both forms at all times
- invite sharing now uses a full share/copy link based on `/game/leagues?invite=CODE`
- `/game/leagues/[id]` now shows:
  leaderboard
  invite share card
  leave action for members
  remove-member and delete-league actions for owners
- `/game/predictions` now hides matches until both teams are resolved in the DB
- joker toggles now disable the rest of the current stage immediately in local UI state

### Corrections To Older Notes

- `/game` still redirects to `/game/predictions`, not `/game/leagues`
- `tournament_predictions.predicted_winner_team_id` is `uuid`, not `bigint`
