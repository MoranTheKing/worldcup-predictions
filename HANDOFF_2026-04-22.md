## Phase 2: League Debug + Full Predictions Hub

Date: 2026-04-22

### Bug Fix: Create League Failing Silently

Root cause: `leagues.owner_id` had TWO foreign key constraints:
1. Original init FK ‚Üí `public.users(id)` (old table)
2. Phase 1 FK ‚Üí `public.profiles(id)` (new table)

New users (Google OAuth) may not exist in `public.users`, causing the insert to fail
violating the original FK. The error was swallowed by the generic error path.

Fixes applied:
- **Migration `20260422000011_fix_leagues_owner_fk.sql`**: drops `leagues_owner_id_fkey`
  (old FK to `users`), backfills `public.users` from `auth.users`, ensures the Phase-1
  profiles FK exists.
- **`app/actions/league.ts` rewrite**: full `try/catch` on every DB call, `console.error`
  with the actual error message, admin client used for the `leagues` insert (bypasses RLS
  edge-cases while auth is still verified via `getUser()`).

## Phase 2.2 - League Route Stabilization + Predictions History

Date: 2026-04-22

This pass fixed a league-route regression and upgraded the match prediction UX into a historical feed.

### League Route Fix

- `/game/leagues/[id]/page.tsx` now loads the league via the admin client
- access is verified explicitly:
  owners are allowed
  members with a `league_members` row are allowed
  authenticated non-members are redirected back to `/game/leagues`
- this prevents false `404` screens right after `createLeague` redirects

### Joker Separation Fix

- joker bucket detection no longer relies on raw `stage === "group"`
- helpers now normalize the stage string first and only then classify:
  group-stage joker
  knockout joker
- optimistic UI now disables only the current bucket instead of the entire match list

### Predictions Feed Upgrade

- `/game/predictions` now renders:
  finished matches
  live matches
  scheduled matches
- future unresolved placeholders are still hidden until both team IDs exist
- finished matches are read-only and show:
  actual result
  user prediction
  `points_earned`
- exact score hits receive premium styling, and exact hits with a joker get the strongest highlight
- the page auto-scrolls to the first `live` or `scheduled` match on initial load

## April 22, 2026 - Emergency RLS Recovery For Prediction Saves

This follow-up was required after a live regression where saving or loading predictions still triggered:

- `infinite recursion detected in policy for relation "league_members"`

### What changed

- `app/game/layout.tsx` now loads gamer-card profile data and joker usage through `createAdminClient()` after the user is authenticated
- `app/game/predictions/page.tsx` now loads:
  prediction history
  tournament outrights
  joker usage
  through the admin client as well
- `app/actions/predictions.ts` still authenticates with the user session first, but match lookup / prediction upsert / tournament upsert now execute through the admin client
- this removes the user-facing dependency on broken legacy RLS during prediction save and page refresh

### New migration

- added: `supabase/migrations/20260422000015_force_flat_prediction_rls.sql`
- purpose:
  drop every policy currently attached to `league_members`, `predictions`, and `tournament_predictions`
  recreate the intended flat authenticated policies from scratch

### What remains
