## Server Actions: `app/actions/league.ts`

Two server actions with full auth guard, error handling, and redirect-on-success.

### `createLeague(formData)`

1. Authenticates the user via `supabase.auth.getUser()`
2. Inserts a new row into `leagues` (`name`, `owner_id`); `invite_code` is auto-generated by the DB trigger
3. Inserts the owner into `league_members`
4. Redirects to `/game/leagues/[id]`

Error cases returned to UI:
- Not authenticated
- Empty league name
- DB insert failure

### `joinLeague(formData)`

1. Authenticates the user
2. Looks up the league by `invite_code` using **admin client** (needed because RLS blocks
   non-members from reading the `leagues` table before they join)
3. Inserts the user into `league_members`
4. Redirects to `/game/leagues/[id]`

Error cases returned to UI:
- Not authenticated
- Empty invite code
- Invalid/unknown invite code (admin lookup returns null)
- PostgreSQL `23505` unique violation Рєњ "ОљОфОћ ОЏОЉОе ОЌОЉОе ОЉОюОЎОњОћ ОќОЋ"
- Generic DB failure
